kidpy is where users can interact with the mkid readout system. Simply launch with .. codeblock:

python kidpy.py

When an operation is selected, a command is created and published on a redis pubsub channel. Any listening RFSOC(s) would then parse and execute the specified command

kidpy.wait_for_reply(redis_pubsub, cmd, max_timeout=15)[source]

This is the eventual replacement for the waitForFree() method. We want to have smarter replies that can ferry data back from the RFSOC.

Once a command is sent out, listen for a reply on the <cmd_reply> channel
Format for replying to commands from redis
message = {

‘cmd’ : ‘relay command’, ‘status’ : ‘OK’|’FAIL’, ‘data’ : ‘nil’ | <arbitrary data>

}

Parameters:

  • max_timeout – int : time in seconds to wait for the RFSOC to reply. If it fails in this time, this should indicate a failure of some kind has occured

  • cmd – str : Command sent out

  • redis_pubsub – redis.Redis.pubsub : reference to a Redis pubsub object that has already subscribed to relevant channels
